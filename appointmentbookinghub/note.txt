
# booking system
- patients book appointments

Patient sees list of specialists
Patient clicks a specialist
Patient sees Specialist's availability calendar
Patient selects a time on Specialist's calendar
Patient sees success page


Specialist sees list of bookings


MODELS

User {
    id
    username
    email
    password
    role: "patient" | "specialist"
    createdAt
}


Patient {
    id
    userId: User.id (OneToOne)
    createdAt

    # derived / reverse
    bookings: Booking[]
}


Specialist {
    id
    userId: User.id (OneToOne)
    specialty
    bio
    createdAt

    # derived / reverse
    bookings: Booking[]
    availabilities: Availability[]
}


Availability {
    id
    specialistId: Specialist.id
    weekday: 0-6 (Mon-Sun)
    startTime
    endTime
}


Booking {
    id
    patientId: Patient.id
    specialistId: Specialist.id

    startTime
    endTime

    status: "pending" | "confirmed" | "cancelled"

    createdAt
}




API

AUTH

POST /register
    body:
    {
        username,
        email,
        password,
        role
    }

POST /login
    body:
    {
        email,
        password
    }



SPECIALISTS

GET /specialists
    - returns list of specialists

GET /specialists/:id
    - returns specialist details

POST /specialists (optional / admin only)
    - create specialist profile


CALENDAR

GET /specialists/:id/calendar?date=YYYY-MM-DD
    - returns available time slots for that date
    - removes the times already booked

Response:
{
    date,
    availableSlots: []
}



BOOKING

POST /bookings
    body:
    {
        specialistId,
        startTime,
        endTime
    }

    - validate specialist exists
    - validate patient role
    - validate time inside availability
    - validate no overlapping booking

    200:
    return Booking
